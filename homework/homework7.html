Task 4:

Write a brief (<200 words) essay outlining the differences between AWT and Swing graphics.

AWT(Abstract Window Toolkit) is an early package which focus on drawing graphs. Actually AWT does NOT draw the graphs itself. It calls the OS GUI library instead. Because it is based on the C/C++ program, it runs fast undoubtedly, but how the graphs that drawn by AWT look like is depends on the platform. So the compatibility is the most important defect of AWT, which obviously goes against with JAVA's design philosophy.

As for Swing, it procides not only all the functions that AWT gives, but also a substantial expansion with pure JAVA code. It runs slower, but gives the same result on all platform. Swing is now more recommended in JAVA.
