Question 1:

For the three categories of web software technology we've just defined, you are asked to find out which category so-called "Netscape plug-in" technology belongs to. Please describe the basic idea of the technology, and give your reasoning on its position in the technology categories.

I think "Netscape plug-in" technology belongs to the third category, namely Distributed Object Technology. This quote below from Wikipedia describes how plugins work:

When the browser encounters a content type it cannot handle natively, it loads the appropriate plugin, sets aside space within the browser context for the plugin to render and then streams data to it.

Obviously this shows the interoperability between the browser and the server in spite of the space interval.

Question 2:

In a normal intercommunication between Web browser and Web Server, Web Server closes the network communication right after it sends back the document requested by the browser. Think about this statusless character of World Wide Web. Give some good points and bad points of it. Can a web server know exactly how many browsers are reading its document?

Good points:
1. This can alleviate the burden of the Server, which increaces the stability.
2. Avoiding invalid network bandwidth occupation.
3. Good for safety.

Bad points:
1. Extra time spends on constructing connections.
2. Redundent Header should be sent between client and server in every connection.
3. Relative low response rate.

My answer is no. Because once the connection is closed, the server has totally no information about what the client is doing now. The only thing a server can do is to record the number of browsers that have download its document.
